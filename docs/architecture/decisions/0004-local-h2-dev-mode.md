# ADR 0004: 本地默认 H2 + 本地文件模式

## 状态

Accepted

## 背景

开发者需要快速启动项目验证 OCR、AI、题图、导出和标准题目包能力。如果本地强依赖 MySQL、Redis、MinIO、MQ，会增加入门成本。

生产环境又必须使用平台级数据库、对象存储、监控和安全边界。

## 决策

本地默认：

- H2 file database。
- Java 本地文件存储。
- Redis/MinIO/MQ 可选。
- Python worker 本地进程。

生产推荐：

- MySQL。
- 对象存储或平台文件中心。
- Redis/MQ。
- Java backend + Python worker 内网部署。

## 后果

正向影响：

- 新开发者可以快速跑通。
- 文档和脚本简单。
- 本地小平台可独立闭环验证。

代价：

- 本地行为不能直接代表生产安全和容量。
- 需要明确交付包和部署手册，避免平台误用 H2 和本地文件。

## 约束

生产不得使用本地 H2 作为主库。

生产文件不得长期落在 Java 本地目录。
