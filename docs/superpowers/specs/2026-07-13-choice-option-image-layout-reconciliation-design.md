# 选择题选项完整性与题图二维归属升级设计

日期：2026-07-13  
状态：已确认方案，待书面规格复核  
目标分支：`codex/image-placement-upgrade`

## 1. 背景与实证

服务器任务 `import_task_20260713_011241_577801d8` 暴露了两类可复现问题：

1. 第 4 题跨页排版，OCR Markdown 将 `D` 粘在 C 的说明文字后，最终只生成 A、B、C 三个选项；第 4 张图被错误归入 C，但 placement 仍得到 `0.99` 置信度。
2. 第 6 题的图片位于选项字母之前。Markdown 顺序把轮胎图留在题干，把骆驼图归入 A，导致 B 无图；两项错误 placement 分别得到 `0.98` 和 `0.99` 置信度。

当前图片 placement 的 `sourceEvidence.pageIndex` 与 `bbox` 为空，而独立的原卷预览已经能从 MinerU `middle.json` 生成跨页题目区域。这说明题图归属阶段没有消费可用的二维版面证据。标准化阶段随后把错误的 `options` 和 `imagePlacements` 当作稳定基线保护，因此增加模型调用无法从根因上修复问题。

## 2. 目标

1. 恢复被 OCR 粘连、漏标或跨页切断的 A–D 选项序列。
2. 使用页码和 bbox 将图片、选项标签与说明文字组成二维选项单元格。
3. 对图片型选择题执行一对一全局分配，避免局部最近邻造成重复占用和空选项。
4. 降低仅由 Markdown offset 得出的虚高置信度。
5. 对不完整选项、题干/选项冲突、图片守恒失败等情况阻止自动写回。
6. 确定性证据不足时才调用多模态模型，并限制模型只能输出归属映射。
7. 支持基于已保存 OCR 产物重新计算结构，不重新运行 MinerU。
8. 保留人工 placement，自动重算不得覆盖 `confirmed` 或 `overridden` 结果。

## 3. 非目标

- 不更换 MinerU。
- 不让多模态模型重写题干、答案或解析。
- 不根据学科答案语义推断正确选项；视觉模型只判断版面归属。
- 不对所有普通文字选择题强制构造四个选项。
- 不自动应用低置信结果。
- 不合并到 `main`，继续在现有分支开发和灰度。

## 4. 方案比较与决策

### 4.1 仅扩充正则

优点是改动小，可以识别 `运动鞋底的鞋钉 D` 等粘连标记。缺点是无法可靠处理图片位于标签之前、双栏、四宫格和跨页单元格，只能作为第一层候选恢复。

### 4.2 二维版面图与全局约束匹配

把图片、A–H 标签、说明文字、题号和页面边界建成图，先生成选项单元格，再使用一对一约束求全局最优归属。该方案可解释、可回归、无需让模型理解全部题目，是本次主方案。

### 4.3 端到端多模态识别

复杂版式适应性较强，但费用、延迟和稳定性不适合全量调用。本次只将其作为确定性结果冲突时的受限兜底。

最终决策：正则候选恢复 + 二维全局匹配为主，多模态仅处理低置信冲突。

## 5. 总体数据流

```text
MinerU Markdown + content_list/middle JSON + 图片资产
    ↓
题目与跨页区域定位
    ↓
宽容选项候选检测与 A–H 序列恢复
    ↓
图片/标签/说明文字二维节点标准化
    ↓
选项单元格构建
    ↓
全局图片归属求解
    ↓
结构不变量与置信度校准
    ↓
高置信自动采用 / 低置信多模态复判 / 人工复核
    ↓
生成 options、images、imagePlacements 和审计摘要
```

图片资产仍由 `images[]` 保存；归属由 `imagePlacements[]` 保存。Markdown 图片引用是渲染结果，不再作为二维归属的唯一事实来源。

## 6. 选项完整性恢复

### 6.1 候选标记

除现有带标点和独立行标签外，新增“期望序列中的弱标签”候选：

- 前面已经出现 A、B、C，文本尾部存在单独字母 D；
- 字母前有空格、换行或短中文说明，字母后紧跟换行、图片或短说明文字；
- 候选位于当前题目区域内，且不跨越下一题题号；
- 候选顺序只能补齐当前期望标签，不能跳过或倒序。

弱标签本身不能直接成为高置信选项，必须获得图片、说明文字、bbox 或完整 A–D 序列中的至少一项补强证据。

### 6.2 序列选择

对同一道题的所有候选生成 A 起始、单调递增的候选链，按以下得分选最佳链：

- 完整 A–D：加分；
- 每个标签后存在非空说明或图片：加分；
- 标签在不同页但题目跨页区域连续：允许且加少量分；
- 标签落在公式、英文单词或普通正文中：扣分；
- 出现重复标签、倒序、越过下一题：淘汰。

普通四选一题识别为 2 或 3 个选项时，不再视为稳定结构；只有题型证据明确且候选链完整，才进入自动图片分配。

## 7. 二维选项单元格

### 7.1 版面节点

从 MinerU 原始产物构造统一节点：

- `question-anchor`：题号和题干起点；
- `option-label`：A–H 标签；
- `text`：选项说明文字；
- `image`：原始图片块；
- `page-boundary`：页码、页宽和页高。

所有图片节点必须带 `pageIndex` 和 bbox。无法从 Markdown 图片路径精确映射到唯一 middle/content 节点时，保持 `unassigned`，禁止按 basename 随机选择。

### 7.2 单元格构建

以选项标签为锚点，将同页的图片和说明文字归入候选单元格。支持三种常见模式：

1. 标签 → 图片 → 说明；
2. 图片 → 标签 → 说明；
3. 标签、图片、说明位于同一行或同一列。

跨页题按页面分别构建单元格，但共享同一 A–D 序列。例如第 4 题允许 A/B 位于第 1 页、C/D 位于第 2 页。

### 7.3 全局归属

对同题图片与选项单元格构造代价矩阵，综合：

- 是否同页；
- 行、列重叠比例；
- 图片到标签及说明文字的归一化距离；
- 阅读顺序；
- 图片位于标签上方、下方或右侧的版式先验；
- 是否造成连线交叉；
- Markdown offset 是否一致；
- 是否造成一个选项多图、另一个选项无图。

默认按一对一约束求最低总代价。只有检测到同一单元格内确有多图聚类时，才允许一个选项对应多图。四图四选项场景优先得到 A–D 的排列，而不是四次独立最近邻。

## 8. 证据优先级与置信度

新的优先级为：

1. 已确认人工 placement；
2. 完整选项链 + 二维单元格 + Markdown 一致；
3. 完整选项链 + 高分二维匹配；
4. Markdown offset；
5. 多模态候选；
6. 未归属。

Markdown offset 不再天然获得 `0.98/0.99`。只有完整序列、无结构冲突且几何一致时，置信度才能达到 `0.95` 以上。

建议阈值：

- `>= 0.95`：自动采用；
- `>= 0.80 且 < 0.95`：保存候选，进入多模态或人工复核；
- `< 0.80`：保持 `unassigned`。

当多个证据冲突时，在 `inference.alternatives` 中保留候选，不静默覆盖。

## 9. 强制结构不变量

下列任一条件触发 `review_required`，并阻止全局标准化自动写回该题：

- 明确四选一但选项数不是 4；
- 选项文本中仍残留疑似下一个标签；
- 图片型选择题存在题干图，同时至少一个选项无图；
- 同一图片拥有多个排他性归属；
- 一个选项多图且另一个选项无图，但没有多图单元格证据；
- placement 缺少 page/bbox，却标记为高置信几何或 offset 结果；
- `images[]`、Markdown 引用和 `imagePlacements[]` 资产不守恒；
- 人工确认结果与自动结果冲突。

`imagePlacementValidation` 必须随题目持久化，并进入标准化 request 的 `structuredHints`。标准化 worker 不得把存在上述阻断项的题判断为规则快速路径或安全写回。

## 10. 多模态兜底

只对置信度 `>= 0.80 且 < 0.95` 或存在二维/offset 冲突的题调用多模态模型。输入限定为：

- 当前题目的局部页图 crop，必要时包含相邻跨页 crop；
- 已检测到的 A–D 标签框；
- 图片框和稳定资产 ID；
- 选项说明文字。

输出必须是受限 JSON，只允许返回 `imageId → stem|A-H|unassigned` 和理由。模型不能修改题干、答案、解析、图片资产或选项数量。结果仍需通过一对一、资产守恒和人工结果保护校验。

多模态不可用、超时或输出非法时保持 `review_required`，不降级为题干图。

## 11. 重新整理与兼容

现有 canonicalization preview/apply 流程扩展为可重新计算选项与题图归属：

- 使用已保存 OCR Markdown、middle/content JSON 和图片，不重新运行 MinerU；
- preview 展示选项数量变化、图片旧归属、新归属、置信度与原因；
- apply 按题目原子写入并保存回滚快照；
- 人工 `confirmed/overridden` placement 默认冻结；
- 旧任务缺少 bbox 时允许生成待复核候选，但不得伪造高置信结果。

不新增第二套前端业务规则。前端只展示后端候选、差异和阻断原因，并继续提供人工调整入口。

## 12. 错误处理与可观测性

每道题记录：

- 检测到的标签链与缺失标签；
- 单元格数量与跨页信息；
- 图片数量、归属数量和未归属数量；
- assignment 总成本、次优成本与 margin；
- offset/geometry/multimodal 冲突数；
- 阻断原因机器码；
- 是否保护了人工结果。

建议机器码包括：

- `choice_option_sequence_incomplete`
- `embedded_option_label_candidate`
- `image_before_first_option`
- `option_image_one_to_one_violation`
- `stem_option_geometry_conflict`
- `missing_image_geometry`
- `cross_page_option_continuation`
- `manual_placement_preserved`

## 13. 测试策略

### 13.1 确定性单元测试

- `A/B/C + “文字 D + 图片”` 恢复 D；
- 图片位于 A 标签之前；
- 四图四选项一对一匹配；
- 双栏、四宫格和纵向列表；
- A/B 在上一页、C/D 在下一页；
- 一个选项合法多图；
- 普通正文中的字母 D 不被误识别；
- 缺少 bbox 时降级为待复核；
- 人工 placement 不被覆盖。

### 13.2 真实样本回归

固化当前服务器任务的第 4、6 题脱敏 fixture，同时保留已有 72 题和四图选择题 fixture。测试必须检查 options 内容、placement 目标、置信度、review 状态和跨页区域。

### 13.3 集成与前端

- canonicalization preview 不修改数据；
- apply 后刷新页面，题干和 A–D 图片位置一致；
- 回滚恢复旧结构；
- 全局标准化跳过或标记阻断题；
- 桌面、中等窗口和窄窗口均显示完整图片与复核原因。

## 14. 验收标准

在至少 600 道带图选择题的审计集上：

- 高置信自动归属 precision `>= 99.5%`；
- 图片型选择题 A–D exact mapping `>= 98%`；
- 选项完整性召回 `>= 99%`；
- 严重跨题误配为 0；
- 缺图、重复归属和悬挂引用各 `<= 0.5%`；
- 所有缺少几何证据的冲突题均进入复核，不再产生虚高置信度；
- 第 4 题恢复 A–D，鞋钉归 C、气垫船归 D；
- 第 6 题轮胎、骆驼、菜刀、图钉分别归 A、B、C、D，题干不再显示轮胎图。

## 15. 发布与回滚

1. 先在保存的 OCR 产物上 shadow 运行，只记录差异。
2. 对真实样本人工审核后按 5% → 25% → 100% 放量。
3. 自动 apply 只允许无阻断、高置信且未覆盖人工结果的题。
4. 保留原 placement、选项和 canonicalization 回滚快照。
5. 发布仍使用 `codex/image-placement-upgrade` 分支，不合并 `main`。

若 exact mapping 或高置信 precision 未达到门槛，关闭自动采用，只保留 preview 与人工复核，不回退到“默认题干图”。
