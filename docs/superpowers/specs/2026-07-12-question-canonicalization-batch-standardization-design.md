# 题目归并、布局框合并与并发标准化设计

日期：2026-07-12
状态：已确认设计，待实施计划

## 1. 背景

当前 OCR 导入链路会把整份 PDF Markdown 中的所有题号锚点都视为题目候选。当 PDF 同时包含试题和“参考答案与试题解析”时，答案区从第 1 题重新编号，可能再次生成一套父题。若 AI 边界确认超时或结构校验失败，流程会回退到本地边界候选，重复题会进入导入任务。

现网任务 `import_task_20260712_143523_4587b484` 提供了可复现证据：

- 原始 Markdown 同时包含试题区和“参考答案与试题解析”区。
- OCR 生成 72 道父题，其中后半段与前半段高度重复。
- 110 个小问中包含答案、分析和最终答案里反复出现的 `(1)(2)(3)`。
- 全局标准化按非空题干、答案、解析和小问字段拆成 225 个内容项。
- 同一道第 2 题被映射为两个不同 `questionId`，并在同一页面形成两个相互重叠的布局框。
- 当前全局标准化由浏览器串行请求，关闭页面会中断，结果在全部处理完之前不会保存。

本设计同时治理三个问题：重复题结构、重复布局框以及不可恢复的串行标准化。

## 2. 目标

1. 将答案区重复出现的题目合并回试题区原题，保留答案、解析和可用的小问信息。
2. 将同一题、同一页的重叠或连续布局框合并为一个可点击区域。
3. 在结构校验通过后，以服务端持久化任务执行并发度为 2 的 AI 标准化。
4. 支持取消、恢复、失败重试、断点保存、幂等和服务重启恢复。
5. 对现有 OCR 任务提供无需重新运行 MinerU 的整理预览、应用和回滚能力。
6. 阻止重复题、异常小问和高风险布局直接进入标准化或批量入库。

## 3. 非目标

- 不在本次升级中更换 MinerU 或大模型供应商。
- 不自动合并无法证明属于同一题的同号题。
- 不把跨页题绘制成一个跨页面巨型框。
- 不在结构整理阶段改写题干语义或重新生成解析。
- 不将并发度直接提高到 4 以上；初始生产并发固定为 2。

## 4. 总体架构

处理链路调整为：

```text
OCR 原始产物
  -> 文档区域识别
  -> 题目候选构建
  -> canonical 题目归并计划
  -> 题目/答案/解析/小问合并
  -> 布局框 canonical 映射与几何合并
  -> 结构质量门禁
  -> 导入任务持久化
  -> 服务端并发标准化批任务
```

实现边界分为四个独立组件：

1. `document_zone_classifier`：识别试题区、答案区和普通附录区。
2. `question_canonicalizer`：生成归并计划并构建唯一题目集合。
3. `layout_region_merger`：按 canonical 题目标识合并同页布局框。
4. `batch_standardization_service`：持久化、调度和恢复标准化批任务。

组件只通过结构化数据通信，不让前端承担题目去重或并发调度责任。

## 5. 文档区域识别

### 5.1 答案区锚点

识别以下独立标题或等价标题：

- 参考答案与试题解析
- 参考答案
- 答案与解析
- 试题答案
- 详解

标题之后出现的题号序列若重新从 1 或较小编号开始，则标记为 `answer-zone`。标题前保持为 `paper-zone`。没有明确答案标题时，不仅凭题号重置判定答案区。

### 5.2 区域证据

每道候选题保留：

- `documentZone`
- `zoneHeading`
- `originalQuestionNumber`
- `sourceEvidence.start/end`
- `sourcePageIndexes`

这些字段用于后续匹配、审计和布局框映射。

## 6. canonical 题目归并

### 6.1 匹配前标准化

比较题干前先生成只用于匹配的 `stemCore`：

- 去除题号和分值前缀。
- 去除结构化选项块，但保留题干正文。
- 去除答案、分析、解答、点评段落。
- 统一空白、全半角标点和常见 OCR 间隔。
- 图片不参与正文字符串，但生成独立图片指纹。

实际题干、选项和 Markdown 不因匹配预处理而被改写。

### 6.2 匹配评分

仅在一个候选属于 `paper-zone`、另一个属于 `answer-zone` 时尝试自动归并。评分由以下证据组成：

- 原始题号一致：0.45
- `stemCore` 相似度：最多 0.35
- 选项标签、选项正文或图片指纹一致：最多 0.10
- 大题类型和章节语义一致：最多 0.10

判定规则：

- 分数不低于 0.85，且同一答案题只有一个最佳候选：自动归并。
- 分数不低于 0.65 且低于 0.85：保留两个候选并标记人工复核。
- 分数低于 0.65：不归并。
- 第一、第二名分差小于 0.08：视为歧义，不自动归并。

题号后缀如 `q_2_2` 仅作为弱提示，不能单独触发归并。

### 6.3 字段合并

归并后以试题区记录作为 canonical 题目：

- 题干、题型、选项、题图优先使用试题区值。
- 答案区的答案和解析补入 canonical 题目。
- 两侧都有非空答案且冲突时，不覆盖，生成 `answer-conflict` 复核项。
- 两侧都有解析时，优先保留明确标记为“解答/解析”的答案区解析；原解析写入审计快照。
- 知识点、难度等元数据只在 canonical 侧为空时补充。
- 所有被归并 ID 写入 `mergedFromQuestionIds`。

### 6.4 小问合并

小问只从题目正文区间提取。遇到“分析、解答、答案、点评”标记后停止创建新小问，答案区的重复 `(1)(2)(3)` 不再成为小问边界。

小问按归一化标签和题干核心匹配：

- 标签相同且题干相似时合并答案与解析。
- 标签重复但正文不同，或标签序列回退时标记复核。
- 单题内自动生成的小问标签必须唯一并保持单调序列。

## 7. 布局框合并

### 7.1 canonical 映射

布局构建前先建立：

```text
sourceQuestionId/importQuestionId -> canonicalQuestionId
```

所有原题和答案区重复题的 region 都先改写为 canonicalQuestionId，再进行几何处理。

### 7.2 同页合并条件

两个 region 只有同时满足以下条件才合并：

1. `pageIndex` 相同。
2. `canonicalQuestionId` 相同。
3. 几何区域相交，或垂直间距不超过页面高度的 2%。
4. 水平方向投影重合比例不低于较窄框宽度的 25%。
5. 两框之间不存在属于其他 canonical 题目的题号锚点。

合并框取最小外接矩形：

```text
x0 = min(left)
y0 = min(top)
x1 = max(right)
y1 = max(bottom)
```

输出记录 `mergedFromRegions`，置信度取参与合并 region 的最低值。

### 7.3 不合并情形

- 同号但属于不同章节且没有 canonical 关系。
- 同一道题跨页出现；每页分别保留一个框。
- 同页框相距过远或中间包含下一题锚点。
- 仅题号相同但题干匹配存在歧义。

## 8. 结构质量门禁

在生成标准化批任务或批量入库前检查：

- 未解决的重复题号。
- `paper-zone` 与 `answer-zone` 未归并的高相似题。
- 小问标签重复、回退或异常膨胀。
- 题目边界结构校验失败。
- canonical 题目与布局框存在高风险映射冲突。
- 题图归属仍为未确认、冲突或需复核。

存在阻断项时，前端显示“请先整理题目结构”，禁止启动全局标准化和批量入库。非阻断警告允许用户逐题复核。

## 9. 现有任务整理与回滚

现有任务使用已保存的 OCR Markdown、JSON、图片和布局产物重建结构，不重新调用 MinerU。

提供两个阶段：

1. 预览：返回原题数、canonical 题数、自动归并项、歧义项、小问变化、布局框变化和阻断项。
2. 应用：保存原结构快照后，原子替换题目结构和 paperLayout。

快照包含任务题目、布局、状态和 canonicalization 版本。每个任务至少保留最近一次应用前快照，并提供回滚入口。预览不写入业务数据。

当前现网任务作为验收样本：第 2 题的 `q_2_2` 应归并回 `q_2`；同页两个重叠框应变为一个框。

## 10. 服务端并发标准化

### 10.1 调度单位

以“题目”为并发单位，而不是以单字段为并发单位：

- 不同题目最多同时处理 2 道。
- 同一道题内按题干、答案、解析、小问顺序处理。
- 一道题全部字段处理结束后，使用一次原子更新保存。
- 单题失败不阻塞其他题目。

这能避免同题多个请求同时写入导致字段覆盖。

### 10.2 持久化模型

新增批任务和任务项持久化：

`ai_standardize_batch_job`：

- `id`
- `task_id`
- `status`: queued/running/cancelling/cancelled/completed/partial_failed/failed
- `total_questions/total_items`
- `completed_questions/completed_items`
- `success_items/failed_items`
- `max_concurrency`
- `created_at/started_at/finished_at`
- `cancel_requested_at`

`ai_standardize_batch_item`：

- `id`
- `job_id`
- `question_id`
- `status`
- `input_hash`
- `attempt_count`
- `item_count`
- `success_count/failed_count`
- `last_error`
- `started_at/finished_at`

现有单次 `ai_job` 继续记录实际模型调用，批任务项引用对应调用 ID，便于审计。

### 10.3 并发与限流

初始配置：

- 批任务题目并发：2
- `LLM_STANDARDIZE_MAX_CONCURRENCY=2`
- `LLM_EXTERNAL_MAX_CONCURRENCY=2`
- 每个导入任务同一时间只允许一个活动标准化批任务

实际并发取调度器、任务信号量和外部端点信号量的最小值。后续只有在超时率和限流率满足运维阈值后才允许提升至 3 或 4。

### 10.4 重试与退避

- 网络超时、429 和可恢复 5xx 最多重试 2 次。
- 退避间隔为 2 秒、5 秒，并尊重上游 `Retry-After`。
- 结构校验失败、候选存在严重公式风险等业务错误不自动重试。
- 重试耗尽后将题目标记为失败，允许仅重跑失败项。

### 10.5 幂等、取消和恢复

- `input_hash` 由 canonicalQuestionId、字段内容和标准化版本生成。
- 相同输入已有成功结果时复用结果，不重复调用模型。
- 每完成一道题立即提交事务并更新进度。
- 取消采用协作式取消：停止领取新题，正在执行的题完成后退出。
- 服务重启后将遗留 `running` 项恢复到 `queued`，依据 input_hash 决定复用或重跑。
- 浏览器只轮询服务端 job，关闭或刷新不会终止任务。

## 11. API 与前端

### 11.1 结构整理 API

- `POST /api/import-tasks/{taskId}/canonicalization/preview`
- `POST /api/import-tasks/{taskId}/canonicalization/apply`
- `POST /api/import-tasks/{taskId}/canonicalization/rollback`

### 11.2 标准化批任务 API

- `POST /api/import-tasks/{taskId}/standardization-jobs`
- `GET /api/import-tasks/{taskId}/standardization-jobs/{jobId}`
- `POST /api/import-tasks/{taskId}/standardization-jobs/{jobId}/cancel`
- `POST /api/import-tasks/{taskId}/standardization-jobs/{jobId}/resume`
- `POST /api/import-tasks/{taskId}/standardization-jobs/{jobId}/retry-failed`

### 11.3 前端交互

结构异常时显示“整理题目结构”入口。预览包含：

- `72 -> N` 道唯一题目的数量变化。
- 自动归并和待复核列表。
- 小问数量变化。
- 布局框合并前后数量。
- 阻断标准化的原因。

标准化进度改为：

```text
已完成 12/36 道题 · 38/95 个内容项 · 并发 2
```

界面提供取消、恢复和仅重试失败项。完成后保留任务摘要，不再在 3 秒后自动清空全部状态。

## 12. 错误处理与可观测性

记录以下指标：

- canonical 自动归并数、歧义数、拒绝数。
- 合并前后父题、小问和布局框数量。
- 标准化队列长度、活动并发、成功率、超时率、429 数。
- 每题和整批耗时。
- 缓存命中和 input_hash 复用次数。
- 取消、恢复和重启恢复次数。

日志不得记录 API 密钥或完整敏感配置。题干只记录哈希和长度，排障详情通过受控 job 记录读取。

## 13. 测试策略

### 13.1 Python 单元测试

- 答案标题后重复编号能匹配原题。
- 普通章节合法重编号不会误判为答案区。
- 去除选项后题干可正确匹配。
- 低分或多候选匹配不自动归并。
- 答案和解析正确补入 canonical 题目。
- 分析、解答和点评中的重复 `(1)(2)(3)` 不生成新小问。
- 同页重叠或紧邻框正确取外接矩形。
- 中间存在其他题号或跨页时不合并。

### 13.2 Java 单元与集成测试

- 批任务同一时刻最多运行 2 道题。
- 同题字段不并发写入。
- 单题完成即持久化。
- 输入哈希可复用成功结果。
- 重试、取消、恢复和服务重启恢复正确。
- 每个任务只能存在一个活动批任务。

### 13.3 前端测试

- 结构门禁阻止异常任务直接标准化。
- 整理预览正确展示题目、小问和框体变化。
- 合并框只渲染一次且点击定位 canonical 题目。
- 进度轮询、取消、恢复和失败重试状态正确。
- 页面刷新后可恢复现有 job 进度。

### 13.4 现网样本回归

使用现有 OCR 产物生成脱敏、小型回归夹具，至少验证：

- `q_2_2` 归并到 `q_2`。
- 第 2 题两个同页框合并为一个框。
- 答案区不再额外生成整套父题。
- 重复分析编号不再膨胀小问数量。
- 标准化队列只包含 canonical 题目及其有效内容项。

## 14. 发布与回滚

1. 在当前 `codex/image-placement-upgrade` 分支实现，不合并 `main`。
2. 默认启用结构预览，应用动作保留人工确认。
3. 服务器部署前备份代码包和业务数据。
4. 先对现有问题任务执行预览，核对归并清单后再应用。
5. 并发度固定为 2，观察超时率、429 和失败率。
6. 若结构归并异常，通过任务快照回滚；若并发异常，将两项并发配置恢复为 1。

## 15. 验收标准

- 答案区重复题不再作为独立题目参与标准化或入库。
- 当前样本第 2 题只显示一个完整布局框。
- 答案、解析、选项、题图和有效小问无数据丢失。
- 合法的章节重编号和跨页题不会被误合并。
- 结构阻断项未解决前无法启动批量标准化。
- 生产标准化并发不超过 2，同一道题不会并发写入。
- 页面刷新、取消和服务重启后任务状态可恢复。
- 失败项可单独重试，成功项不会重复调用模型。
- 当前任务可以从结构快照完整回滚。
- Python、Java、前端和部署 smoke 测试全部通过后方可更新服务器版本。
